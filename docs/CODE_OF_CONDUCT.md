# Code of Conduct

This project aims to be useful, welcoming, and easy to collaborate on. Participation in issues, pull requests, and review discussions should help move the software forward without making the project hostile to contributors or users.

## Expected behavior

- Be respectful, direct, and constructive.
- Focus criticism on code, design, and behavior, not on people.
- Assume good intent, but be willing to correct mistakes clearly.
- Keep technical disagreement evidence-based and relevant to the change being discussed.
- Respect privacy. Do not publish secrets, personal data, or private correspondence.

## Unacceptable behavior

- Harassment, threats, intimidation, or discrimination.
- Personal attacks, insults, dogpiling, or repeated bad-faith engagement.
- Doxxing, stalking, or sharing private information without permission.
- Spam, off-topic disruption, or intentionally derailing reviews and issues.
- Publishing credentials, tokens, private logs, or exploit details in public when a private reporting path is appropriate.

## Enforcement

Maintainers may edit, hide, lock, or remove content that violates this policy. They may also reject contributions or block further participation when necessary to protect the project and its community.

## Reporting

If you experience or observe behavior that violates this code of conduct, contact a maintainer privately through GitHub rather than escalating in public. If the issue involves abuse on GitHub itself, you can also use GitHub's built-in reporting tools.
